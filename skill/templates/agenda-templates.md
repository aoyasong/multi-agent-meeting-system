# 议程模板库

## 通用模板

### 标准会议议程

```markdown
1. 开场（2分钟）
   - 会议目的说明
   - 参会者介绍

2. 议题清单（主体）
   - 议题1：{名称} - {类型：讨论/决策/同步}
   - 议题2：{名称} - {类型}
   - 议题3：{名称} - {类型}

3. 总结与下一步（3分钟）
   - 决议汇总
   - 行动项确认

4. 自由提问（5分钟）
```

### 问题解决议程

```markdown
1. 问题陈述（5分钟）
   - 问题描述
   - 影响范围
   - 紧急程度

2. 根因分析（10分钟）
   - 可能原因收集
   - 根因确认

3. 解决方案讨论（15分钟）
   - 方案1：{描述}
   - 方案2：{描述}
   - 方案对比

4. 决策与行动（5分钟）
   - 方案选定
   - 执行计划
   - 责任人分配
```

## 场景模板

### 头脑风暴模板

```markdown
1. 开场（1分钟）
   - 主题说明
   - 规则：数量优先、不批判

2. 发散阶段（15分钟）
   - 自由发言
   - 实时记录

3. 收敛阶段（10分钟）
   - 归类整理
   - 优先级投票

4. 输出（5分钟）
   - Top N 清单
   - 下一步建议
```

### 需求评审模板

```markdown
1. 开场（2分钟）
   - 评审维度说明
   - 表决规则

2. 需求陈述（5分钟）
   - 需求背景
   - 功能要点

3. 分项审议（20-40分钟）
   - 功能1审议
   - 功能2审议
   - ...

4. 综合评审（10分钟）
   - 汇总待定项
   - 整体决议投票

5. 输出（5分钟）
   - 决议宣布
   - 行动项
```

### 技术评审模板

```markdown
1. 开场（2分钟）
   - 评审维度说明
   - 评分标准

2. 方案陈述（10分钟）
   - 技术方案介绍
   - 关键决策说明

3. 多维评审（30分钟）
   - 架构合理性
   - 可扩展性
   - 性能评估
   - 安全性
   - 可维护性

4. 风险汇总（10分钟）
   - 高风险项讨论
   - 应对方案

5. 决议投票（5分钟）
   - 通过/修改/驳回
```

### 项目启动模板

```markdown
1. 开场与背景（5分钟）
   - 项目来源
   - 业务价值

2. 目标与范围（10分钟）
   - 目标定义
   - 范围边界

3. 团队分工（10分钟）
   - 角色确认
   - 职责明确

4. 里程碑规划（10分钟）
   - 阶段划分
   - 关键节点

5. 风险识别（5分钟）
   - 风险收集
   - 预案制定

6. 协作机制（5分钟）
   - 沟通方式
   - 会议节奏

7. 启动确认（5分钟）
   - 投票确认
   - 行动项
```

### 决策会议模板

```markdown
1. 决策背景（5分钟）
   - 需要决策的问题
   - 决策紧迫性
   - 决策影响范围

2. 选项呈现（10分钟）
   - 选项A：{描述} - 优缺点
   - 选项B：{描述} - 优缺点
   - 选项C：{描述} - 优缺点

3. 讨论与质询（15分钟）
   - 选项对比
   - 疑问澄清
   - 利益权衡

4. 决策投票（5分钟）
   - 投票方式：{简单多数/加权/共识}
   - 结果宣布

5. 执行计划（5分钟）
   - 决策执行步骤
   - 责任人
   - 时间线
```

## 议题类型标记

| 标记 | 类型 | 处理方式 |
|------|------|----------|
| 📢 | 信息同步 | 单向陈述，确认理解 |
| 💬 | 开放讨论 | 自由发言，总结要点 |
| 🗳️ | 决策投票 | 讨论+投票，执行决议 |
| ⚠️ | 风险预警 | 评估风险，制定预案 |
| 🎯 | 目标对齐 | 确认共识，明确方向 |

## 时间分配建议

| 会议时长 | 开场 | 主体讨论 | 总结 | 自由提问 |
|----------|------|----------|------|----------|
| 15分钟 | 1分钟 | 10分钟 | 2分钟 | 2分钟 |
| 30分钟 | 2分钟 | 20分钟 | 5分钟 | 3分钟 |
| 60分钟 | 3分钟 | 45分钟 | 7分钟 | 5分钟 |
| 90分钟 | 5分钟 | 70分钟 | 10分钟 | 5分钟 |

## 议程设计原则

1. **聚焦原则**：每个议程服务于会议目标
2. **时间可控**：为每个议题预估时长
3. **优先级排序**：重要议题前置
4. **类型标注**：明确每个议题的处理方式
5. **弹性缓冲**：预留10%机动时间
