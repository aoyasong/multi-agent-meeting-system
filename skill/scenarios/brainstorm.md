# 头脑风暴场景

## 场景说明

**适用情况**：
- 需要收集大量创意想法
- 探索性话题，没有固定答案
- 团队共创、命名、方案发散等

**预期产出**：
- 创意清单（分类整理）
- 优先级排序（投票结果）
- 可执行方案筛选

## 会议参数建议

```json
{
  "type": "brainstorm",
  "votingComplexity": "simple",
  "speakingMode": "free",
  "estimatedDuration": "15-30 minutes"
}
```

## 标准流程

### 1. 开场（1分钟）

主Agent发言：
> 我们来进行一场头脑风暴，主题是：**{主题}**
>
> 规则很简单：
> 1. 自由发言，数量优先
> 2. 不批判，先记录
> 3. 后续会进行归类和优先级投票
>
> 参会者：{参会者列表}
> 预计时长：{N}分钟
>
> 开始！

### 2. 发散阶段（10-15分钟）

**操作**：
1. 开放自由发言模式
2. 实时收集并展示想法
3. 鼓励补充和延伸

**话术**：
- "还有其他想法吗？"
- "这个角度很有意思，可以展开说说"
- "我们先记下来，稍后归类"

**记录方式**（使用 `recording_take_note`）：
```
想法1: [内容] - 提出者: [Agent名]
想法2: [内容] - 提出者: [Agent名]
...
```

### 3. 收敛阶段（5-8分钟）

#### 3.1 归类整理

主Agent发言：
> 发散阶段结束，我们收集到了 {N} 个想法。
>
> 我来归类整理一下：
>
> **类别A - [领域]**
> - 想法1: ...
> - 想法3: ...
>
> **类别B - [领域]**
> - 想法2: ...
> - 想法5: ...
>
> 有需要调整的吗？

#### 3.2 优先级投票

**创建投票**：
```
voting_create(
  meetingId,
  topic="优先级排序",
  options=["想法1", "想法2", "想法3", ...],  // 或按类别投票
  complexity="simple"
)
```

**投票说明**：
> 请每人选出最值得推进的3个想法，简单投票，窗口3分钟。

#### 3.3 结果宣布

主Agent发言：
> 投票结果：
> 🥇 想法X - {N}票
> 🥈 想法Y - {N}票
> 🥉 想法Z - {N}票
>
> 接下来建议优先推进这些方向。

### 4. 结束与输出

**生成纪要**：
```
## 头脑风暴纪要

**主题**：{主题}
**时间**：{时间}
**参会者**：{列表}

### 收集的想法（共N个）

**类别A**
- 想法1: ...
- 想法3: ...

**类别B**
- 想法2: ...

### 优先级投票结果

| 排名 | 想法 | 票数 |
|------|------|------|
| 1 | 想法X | N |
| 2 | 想法Y | N |

### 下一步行动

1. [ ] 推进想法X的验证 - 负责人: ?
2. [ ] 进一步调研想法Y - 负责人: ?
```

**结束发言**：
> 头脑风暴会议结束。共创了 {N} 个想法，投票选出了前3个优先方向。
> 纪要已生成，是否需要同步到项目文档？

## 示例对话

**用户**：我们来头脑风暴一下新产品的命名

**主Agent**：
> 创建会议中...
>
> 会议已创建：**新产品命名头脑风暴**
> 参会者：产公公、技公公、设公公
> 
> 我们来收集命名创意，自由发言，数量优先！

**参会者A**：我觉得可以叫"OpenBrain"

**参会者B**："ClawMind"怎么样？跟品牌呼应

**参会者C**："智爪" 或者 "ClawAI"

**主Agent**：（记录中）
> 收集到4个命名：
> 1. OpenBrain - 提出者: A
> 2. ClawMind - 提出者: B
> 3. 智爪 - 提出者: C
> 4. ClawAI - 提出者: C
>
> 还有补充吗？

**参会者A**：再加个"BrainClaw"

**主Agent**：
> 发散阶段结束，共5个命名候选。
>
> 启动投票，每人选最中意的2个。

**... 投票进行中 ...**

**主Agent**：
> 投票结果：
> 🥇 ClawMind - 3票
> 🥈 智爪 - 2票
>
> 建议采用 "ClawMind" 作为产品名称。
> 纪要已生成。

## 变体场景

### 命名场景
- 单轮投票即可
- 可设置"每人可提名+投票"规则

### 方案发散
- 需要明确评估维度
- 可能需要多轮投票（维度淘汰制）

### 问题诊断
- 先收集问题，再分类归纳
- 可按"紧急度×重要性"矩阵排序
